# ROI and Pilot Design

The best pilot is not the one with the flashiest demo. It is the one that exposes the real integration burden, operator response, maintenance demand, and value creation pattern early enough to improve the rollout strategy.

## A strong pilot usually has

- A narrow application scope with visible pain and measurable improvement potential.
- Enough variability to be representative, but not so much that every failure is ambiguous.
- Clear ownership for operations, engineering, and maintenance.

## Weak ROI models often fail because

- They assume uptime and adoption without testing them.
- They ignore support, recovery, or changeover burden.
- They overvalue labor displacement while undervaluing process stability and quality impact.
