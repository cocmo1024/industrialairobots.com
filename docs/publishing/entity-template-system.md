# Industrial AI Robots Template System

This repository is designed for AI-operated publishing, but pages must behave like deployment references for manufacturers and integrators, not light thought pieces. Readers should be able to use them to evaluate applications, build pilots, and avoid expensive rollout mistakes.

## Publishing bar

Every substantive page must do all of the following:

1. Target a real automation decision with labor, throughput, safety, or rollout consequences.
2. Make the fit conditions and poor-fit conditions explicit.
3. Explain what must be true for the deployment to work outside a demo.
4. Cover cell, sensing, staffing, maintenance, or change-management implications.
5. Link into the rest of the decision path across application, robot type, cell design, deployment, and case study pages.

General depth targets:

- `seed`: usually 1,200+ words
- `growing`: usually 1,500+ words
- `cornerstone`: usually 1,800+ words

Do not publish:

- trend pieces that do not help an operations team act
- ROI pages without assumptions and failure points
- application pages that ignore fixturing, recovery, or operator reality
- robot-type pages that only compare headline specs

## Universal long-form structure

Use this shape unless there is a strong reason not to:

1. `Quick answer`
2. `When this page should guide the decision`
3. `What makes the application or rollout hard`
4. `Technical and operational design logic`
5. `Failure modes and poor-fit cases`
6. `Pilot or rollout validation`
7. `Implementation checklist`
8. `Related pages to read next`

## Application template

Use for job-focused pages like palletizing, machine tending, inspection, and picking.

Required sections:

1. Quick answer and fit conditions
2. Business case and process reality
3. Robot, gripper, or sensing implications
4. Cell-design and operator-recovery implications
5. Validation metrics for a pilot
6. Failure modes and poor-fit cases
7. Compare next

## Case study template

Use for pilot, rollout, and implementation-pattern pages.

Required sections:

1. What the project pattern is
2. What the pilot or rollout needed to prove
3. Constraints and hidden failure points
4. What changed operationally
5. Why the outcome matters for future buyers
6. Related deployment pages

## Robot type template

Use for cobots, articulated robots, SCARA, gantry, and similar categories.

Required sections:

1. Where the robot class fits
2. Which application assumptions must be true
3. Operational tradeoffs and integration costs
4. Where it is commonly overused
5. Deployment consequences and compare next

## Deployment template

Use for ROI, pilot design, readiness, maintenance, and change-management pages.

Required sections:

1. Decision objective
2. What should be validated early
3. Why ROI models fail in practice
4. Organizational readiness and maintenance ownership
5. Go / no-go criteria
6. Next-step pages
